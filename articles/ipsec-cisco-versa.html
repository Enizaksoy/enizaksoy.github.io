Overview

IPsec tunnel configuration between Cisco routers and Versa SD-WAN appliances for secure hybrid connectivity. This guide covers establishing encrypted tunnels between traditional Cisco infrastructure and modern SD-WAN deployments.

Cisco Configuration

! IKEv2 Proposal
crypto ikev2 proposal VERSA-PROPOSAL
 encryption aes-cbc-256
 integrity sha256
 group 14

! IKEv2 Policy
crypto ikev2 policy VERSA-POLICY
 proposal VERSA-PROPOSAL

! IKEv2 Keyring
crypto ikev2 keyring VERSA-KEYRING
 peer VERSA-PEER
  address <VERSA_PUBLIC_IP>
  pre-shared-key <YOUR_PSK>

! IKEv2 Profile
crypto ikev2 profile VERSA-PROFILE
 match identity remote address <VERSA_PUBLIC_IP>
 authentication remote pre-share
 authentication local pre-share
 keyring local VERSA-KEYRING

! IPsec Transform Set
crypto ipsec transform-set VERSA-TRANSFORM esp-aes 256 esp-sha256-hmac
 mode tunnel

! IPsec Profile
crypto ipsec profile VERSA-IPSEC-PROFILE
 set transform-set VERSA-TRANSFORM
 set ikev2-profile VERSA-PROFILE

! Tunnel Interface
interface Tunnel0
 ip address 169.254.1.1 255.255.255.252
 tunnel source GigabitEthernet0/0
 tunnel destination <VERSA_PUBLIC_IP>
 tunnel mode ipsec ipv4
 tunnel protection ipsec profile VERSA-IPSEC-PROFILE

Key Considerations

  • NAT Traversal: Enable NAT-T if either endpoint is behind NAT
  • DPD (Dead Peer Detection): Configure keepalives for tunnel health monitoring
  • PFS (Perfect Forward Secrecy): Recommended for enhanced security
  • MTU: Account for IPsec overhead (typically 1400 bytes)
  • Routing: Configure static routes or routing protocol over tunnel
